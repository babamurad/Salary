### Как сделать проект Open Source (Пошаговый план)

Просто нажать кнопку "Сделать публичным" в GitHub недостаточно.
Чтобы проект считался настоящим Open Source и выглядел профессионально, выполните эти 3 шага:

**Шаг 1. Добавьте файл `.gitignore` (Критически важно!)**
В папке с исходниками Delphi генерируется куча мусора при компиляции (файлы `.dcu`, `.exe`, `*.~*`, `__history`).
Их **нельзя** заливать в GitHub!
Создайте в корне проекта файл с названием `.gitignore` (с точкой в начале) и пропишите туда исключения.

**Шаг 2. Выберите и добавьте Лицензию (License)**
Без лицензии ваш код защищен авторским правом, и по закону другие люди не могут его
использовать или менять, даже если репозиторий открыт.
1. На странице вашего репозитория в GitHub нажмите **Add file -> Create new file**.
2. Назовите файл **`LICENSE`** (заглавными буквами).
3. Справа появится кнопка **Choose a license template**.
4. Выберите **MIT License** (это самая популярная и свободная лицензия, разрешающая
делать с кодом всё что угодно). Нажмите *Commit new file*.

**Шаг 3. Сделайте репозиторий публичным**
Если вы изначально создали его как Private:
1. Зайдите в ваш репозиторий на GitHub.
2. Перейдите во вкладку **Settings** (Настройки).
3. Прокрутите в самый низ до секции **Danger Zone** (Опасная зона).
4. Найдите пункт **Change repository visibility** и нажмите **Change visibility**.
5. Выберите **Change to public**, подтвердите действие
(GitHub попросит ввести название репозитория для проверки).
