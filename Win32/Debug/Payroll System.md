# 🗺️ План модернизации Payroll System (Версия 2.0)

## Этап 1: Архитектура базы данных (Фундамент)
Без правильных таблиц мы не сможем реализовать логику. Нам предстоит:

*   **Добавить в таблицу `employees` новые поля:** тип графика (стандарт/вахта/сменный), тип оплаты (оклад/часовой тариф), индивидуальный % пенсионного взноса.
*   **Спроектировать совершенно новую таблицу `timesheet` (Табель)**, которая будет хранить привязку: `Сотрудник -> Дата -> Отработанные часы -> Код статуса` (например, **Я** — явка, **В** — выходной, **Р** — ремонт для водителей, **ВМ** — вахтовый метод).

---

## Этап 2: Обновление интерфейса «Сотрудники» и Поиск

*   **Доработаем карточку сотрудника**: выведем удобные выпадающие списки (`ComboBox`) для выбора типа оплаты и графика.
*   **Добавим поле для ввода** индивидуального пенсионного процента.
*   **Реализуем «умный поиск»**: мгновенное нахождение человека как по ФИО, так и по табельному номеру.

---

## Этап 3: Создание модуля «Табель рабочего времени» (Самый объемный)

*   **Новый визуальный интерфейс (фрейм)**, где кадровик или табельщик сможет выбрать месяц, отдел и заполнить часы по дням.
*   **Автоматизация**: система автоматически подставляет норму по производственному календарю, а пользователь меняет только отклонения (например, ставит «Р» — ремонт для водителей).

---

## Этап 4: Новое ядро расчета зарплаты (Математика)
Перепишем процедуру начисления. Теперь она будет брать не просто «оклад за месяц», а смотреть в Табель:

*   **Для стандартных**: сверять отработанные дни с нормой.
*   **Для часовых/сменных**: умножать тариф на количество часов.
*   **Для водителей**: отдельно считать часы работы по одному тарифу и часы простоя (ремонта) по другому.
*   **Пенсионный взнос**: внедрение вычета индивидуального пенсионного взноса при финальном расчете.
