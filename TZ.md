Техническое задание на разработку программы «Зарплата ТМ» (Версия 2.0)
1. Цель проекта
Разработка автоматизированного рабочего места (АРМ) бухгалтера для расчета заработной платы сотрудников в Туркменистане с возможностью гибкой настройки налоговых ставок и ведения истории начислений.

2. Стек технологий

Среда разработки: Delphi (Object Pascal).
+2


База данных: SQLite (локальный файл salarydb.db).
+3


Движок доступа к данным: FireDAC (TFDConnection, TFDQuery).
+1

Архитектура: Основная расчетная логика вынесена на сторону СУБД (SQL Views).

3. Архитектура базы данных
3.1. 
Основные таблицы 
employees: Реестр сотрудников.

Поля: id, fio (ФИО), hire_date (дата приема), base_salary (оклад), status (1 — активен, 0 — уволен).

const_settings: Справочник констант.

Поля: key_name (название параметра), key_value (значение).

Ключевые параметры: tax_rate (10%), pension_rate (2%).

payroll_journal: Архив выплаченной зарплаты.

Хранит фиксированные суммы на момент начисления, чтобы изменение налогов в будущем не меняло историю прошлых периодов.

3.2. 
Расчетный слой (SQL Views) 
Программа должна использовать представление v_current_payroll для динамического отображения текущих сумм «на руки».

Формула пенсионного взноса: base_salary * (pension_rate / 100).

Формула подоходного налога: (base_salary - пенсионный) * (tax_rate / 100).

4. Функциональные требования
4.1. Модуль учета кадров
Добавление, редактирование и увольнение сотрудников.

Автоматическое определение стажа для последующего расчета больничных.

4.2. Модуль гибких настроек
Интерфейс для изменения ставок подоходного налога и пенсионного фонда.

Ввод необлагаемого налогом минимума (согласно законодательству ТМ).

4.3. Учет рабочего времени и отсутствие
Производственный календарь: Возможность пометки праздничных и перенесенных дней.

Больничные: Расчет суммы пособия в зависимости от стажа (настраиваемая шкала процента выплат).

Отпуска: Расчет среднего заработка за последние 12 месяцев.

4.4. Формирование отчетности
Генерация расчетного листка для каждого сотрудника.

Формирование сводной ведомости за месяц для экспорта.

5. Программная реализация в Delphi
5.1. 
Модуль данных (DataModule) 
Использование компонента TFDConnection с динамической настройкой пути к БД в событии BeforeConnect.
+1

Применение TFDQuery для работы с SQL-представлениями.

5.2. Обработка ошибок и надежность
Использование SQL-транзакций при записи данных в журнал начислений.

Валидация вводимых данных (запрет отрицательных окладов).

6. Требования к интерфейсу
Главное окно: Сетка (TDBGrid) со списком сотрудников и текущими расчетами из v_current_payroll.

Панель настроек: Поля ввода для изменения глобальных коэффициентов.

Календарь: Визуальный выбор праздничных дней.
